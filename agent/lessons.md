# Lessons Learned

> **上限**: 15エントリ / 120行。肥大化はコンテキストウィンドウを圧迫し全作業品質を下げる。
> **卒業**: 10件到達時に類似統合 →「カスタマイズファイルの選択基準」表に従い適切なカスタマイズファイルへの昇格を判断する。

## 2026-03-01: assert_true は bash キーワードと非互換

- **パターン**: `assert_true [[ "$var" == "yes" ]]` が失敗する
- **問題**: `assert_true` は `"$@"` で受け取った引数を実行するが、`[[ ]]` は bash キーワード（コマンドではない）ため、コマンドとして実行できない
- **ルール**: `assert_true` には外部コマンドまたは関数のみ渡す。条件式は `assert_eq` に変換するか、`test` コマンド（`[` の方）を使う

## 2026-03-01: Docker ソケット GID ≠ docker グループ GID

- **パターン**: `detect_docker_gid` のテストで GID が期待値と異なる
- **問題**: Docker ソケットの GID（`stat` で取得）とホストの `docker` グループ GID（`getent` で取得）は異なる場合がある。Dev Container 内では特にずれる（例: ソケット GID=989, グループ GID=1001）
- **ルール**: GID のハードコードは行わない。検出ロジックの優先順位（ソケット stat > rootless > getent）を維持し、フォールバックは「失敗」とするのが正しい

## 2026-03-01: テスト統合時は CI パイプラインも同時更新

- **パターン**: test.sh → tests/run_all.sh への移行
- **問題**: テストファイルを移動・統合した際に CI（`.github/workflows/ci.yml`）の更新を忘れると、CI がパスしなくなる
- **ルール**: テスト構造変更時は必ず CI 設定の `chmod` と実行パスを同時に変更する

## 2026-03-03: awk -v はバックスラッシュをエスケープとして解釈する

- **パターン**: `awk -v var="$content"` で `\` + 改行を含む Dockerfile テンプレートを渡す
- **問題**: `awk -v` は C スタイルのエスケープ解釈を行い、`\` が消える。CI 環境の awk バージョンで特に顕著
- **ルール**: バックスラッシュを含む文字列は `ENVIRON["VAR"]` 経由で渡す。対応して `\\\\` → `\\` にエスケープ数を調整する

## 2026-03-03: 固定値で十分な設定を設定ファイル化しない

- **パターン**: generate-workspace.sh のスキャン対象ディレクトリを `workspace.toml` の `[workspace].paths` で設定可能にした
- **問題**: コンテナ内では常にスクリプトの親ディレクトリがスキャン対象で固定。設定可能にする必要がない箇所を設定化すると過剰設計になる
- **ルール**: 動作環境で値が固定される場合は、設定ファイルに切り出さずハードコード（`PARENT_DIR`）のままにする

## 2026-03-01: ボリュームマウント先ディレクトリは必ずユーザー権限で事前作成

- **パターン**: Dockerfile.template の `mkdir -p` を一括削除してインストール関数に移動
- **問題**: ボリュームマウント先が存在しないと Docker が root 権限でディレクトリを作成し、コンテナ内ユーザーがアクセスできなくなる
- **ルール**: ボリュームマウントするディレクトリは、対応する `generate_*_install` 関数内で `USER ${USERNAME}` の状態で `mkdir -p` すること。インストールとディレクトリ作成はセットで管理する
