# Lessons Learned

> **上限**: 10エントリ / 100行。肥大化はコンテキストウィンドウを圧迫し全作業品質を下げる。
> **卒業**: 1週間以上残存するエントリは instructions/skills への昇格を判断する。

## 2026-03-01: assert_true は bash キーワードと非互換

- **パターン**: `assert_true [[ "$var" == "yes" ]]` が失敗する
- **問題**: `assert_true` は `"$@"` で受け取った引数を実行するが、`[[ ]]` は bash キーワード（コマンドではない）ため、コマンドとして実行できない
- **ルール**: `assert_true` には外部コマンドまたは関数のみ渡す。条件式は `assert_eq` に変換するか、`test` コマンド（`[` の方）を使う

## 2026-03-01: Docker ソケット GID ≠ docker グループ GID

- **パターン**: `detect_docker_gid` のテストで GID が期待値と異なる
- **問題**: Docker ソケットの GID（`stat` で取得）とホストの `docker` グループ GID（`getent` で取得）は異なる場合がある。Dev Container 内では特にずれる（例: ソケット GID=989, グループ GID=1001）
- **ルール**: GID のハードコードは行わない。検出ロジックの優先順位（ソケット stat > rootless > getent）を維持し、フォールバックは「失敗」とするのが正しい

## 2026-03-01: テスト統合時は CI パイプラインも同時更新

- **パターン**: test.sh → tests/run_all.sh への移行
- **問題**: テストファイルを移動・統合した際に CI（`.github/workflows/ci.yml`）の更新を忘れると、CI がパスしなくなる
- **ルール**: テスト構造変更時は必ず CI 設定の `chmod` と実行パスを同時に変更する
